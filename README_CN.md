# Tretra Trading Station - 快速使用指南

## 平台简介

Tretra Trading Station 是一个专业的加密货币交易分析平台，集成两大核心功能：

1. **山寨币筛选器** - 实时筛选高Beta机会，捕捉强势山寨币
2. **交易回测分析** - 导入历史交易数据，深度分析和可视化

## 一键启动（推荐使用Docker）

### Docker部署（推荐）

```cmd
cd C:\Users\atxin\Desktop\binance-altcoin-screener

# 一键启动所有服务
docker-compose up -d

# 访问平台
打开浏览器: http://localhost:3000
```

启动后，在顶部导航栏可切换：
- **山寨币筛选器** - 实时筛选功能
- **交易回测分析** - 导入CSV分析历史交易

## 本地开发启动

### Windows用户

1. **安装Python环境** (如果未安装)
   - 下载 Python 3.8+ from https://www.python.org/downloads/
   - 安装时勾选 "Add Python to PATH"

2. **安装依赖**
   ```cmd
   cd C:\Users\atxin\Desktop\binance-altcoin-screener
   pip install -r backend\requirements.txt
   ```

3. **配置API密钥**
   - 复制 `.env.example` 为 `.env`
   - 填入你的币安API密钥

4. **启动服务**
   ```cmd
   # 启动后端
   python run_backend.py

   # 新开一个命令行窗口，启动监控 (可选)
   python run_monitor.py
   ```

5. **访问Web界面**
   - 打开浏览器访问 http://localhost:8000

### 仅命令行使用

如果你不需要Web界面，只想运行自动监控:

```cmd
python run_monitor.py
```

这将每5分钟自动筛选一次，发现高分机会时通过邮件/Telegram通知你。

## 交易回测分析使用

### 导入交易数据

1. 从币安/OKX下载历史交易CSV文件
2. 在平台顶部点击"交易回测分析"
3. 上传CSV文件
4. 系统自动分析并生成可视化报告

### 支持的数据格式

- **币安现货/合约交易数据**
- **OKX交易数据**
- 自动识别数据格式

### 分析指标

- ✅ 胜率、盈亏比
- ✅ 最大回撤、夏普比率
- ✅ 累计收益曲线
- ✅ 按币种、按时段统计
- ✅ 持仓时长分析
- ✅ Excel导出功能

## 主要参数说明

### 时间周期
- **5m**: 适合捕捉短期价格异动
- **15m**: 平衡短期和中期信号
- **1h**: 过滤噪音，关注较大趋势
- **4h**: 中长期趋势分析

### 最小成交量
- 建议设置 1,000,000 USD (100万美元)
- 过低容易陷入流动性不足的币种
- 过高会错过小市值潜力币

### 评分说明
- **80+分**: 强烈信号，多个指标共振
- **70-80分**: 良好机会，值得关注
- **60-70分**: 一般机会，需要额外确认
- **60分以下**: 信号较弱

## 使用技巧

### 1. 结合BTC/ETH趋势
当BTC或ETH处于明显上升趋势时，使用本工具效果最佳。在震荡或下跌市场，Beta策略可能失效。

### 2. 多时间周期确认
不要只看单一时间周期，建议:
- 5m/15m 用于进场时机
- 1h/4h 用于确认大趋势

### 3. 注意成交量
即使评分很高，如果成交量不足，也要谨慎。建议24h成交量至少200万美元以上。

### 4. 及时止损
工具只提供筛选和信号，不保证盈利。请设置合理止损。

### 5. 分散投资
不要把所有资金投入单一币种，建议分散到3-5个高分标的。

## 常见问题

**Q: 为什么筛选结果为空?**
A: 可能是:
1. 筛选条件过于严格
2. 当前市场没有符合条件的机会
3. 币安API连接问题

解决: 降低最小成交量或最小变动百分比

**Q: 图表生成很慢?**
A: 正常现象，生成图表需要:
1. 下载500根K线数据
2. 计算技术指标
3. 渲染图表
通常需要5-10秒

**Q: 可以用其他交易所吗?**
A: 当前版本仅支持币安现货。如需支持其他交易所，需要修改代码。

**Q: 历史数据如何清理?**
A: 数据库文件在 `data/screener.db`，可以直接删除重新生成，或使用SQL清理旧数据。
